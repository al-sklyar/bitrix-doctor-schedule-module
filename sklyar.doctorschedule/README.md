# sklyar.doctorschedule

`sklyar.doctorschedule` - тестовый модуль для 1C-Bitrix с публичной записью на прием, справочниками врачей и услуг, расписанием, ценами и таблицей записей.

Важно: это не готовый production-модуль, а тестовое задание. Решение подходит для демонстрации архитектуры, установки, админки и публичного сценария записи, но не должно восприниматься как финальный медицинский продукт без дальнейшей доработки.

## Что делает модуль

Модуль:

- создает HL-блоки со справочниками;
- создает таблицы расписания, цен и записей;
- добавляет административные страницы для управления таблицами модуля;
- устанавливает публичный компонент `sklyar:doctorschedule.booking`;
- поддерживает два шаблона публичного компонента: `.default` и `compact`.

## Установка

1. Поместить модуль в папку:

```text
/local/modules/sklyar.doctorschedule/
```

2. В административной части Bitrix открыть установку модулей и установить `sklyar.doctorschedule`.

3. На шаге установки при необходимости включить опцию:

```text
Заполнить тестовыми данными?
```

Если галка включена, модуль создаст тестовые записи в справочниках и таблицах, чтобы можно было сразу проверить админку и публичную запись.

4. После установки модуль копирует публичный компонент в:

```text
/local/components/sklyar/doctorschedule.booking/
```

## Настройки модуля

Сейчас в `options.php` добавлены только настройки, связанные с персональными данными:

- заголовок попапа с согласием;
- текст согласия на обработку персональных данных.

Эти значения используются в публичной форме записи на прием. Без установленной галки согласия запись не создается.

## Какие HL-блоки создаются

Модуль создает 5 HL-блоков:

- Врачи
- Специализации
- Типы приема
- Локации
- Услуги

Управление этими справочниками выполняется через стандартный интерфейс Highload-блоков Bitrix.

## Какие таблицы создаются

Модуль создает 3 таблицы:

- `sklyar_ds_schedule_rule` - правила расписания врача;
- `sklyar_ds_service_price` - цены и параметры услуг;
- `sklyar_ds_booking` - записи на прием.

## Административные разделы

Для управления таблицами модуль добавляет раздел:

```text
Сервисы -> Расписание врача
```

Внутри доступны страницы:

- `Правила расписания` - управление таблицей `sklyar_ds_schedule_rule`. Здесь можно добавлять, редактировать и удалять интервалы приема по врачам и дням недели.
- `Услуги и цены` - управление таблицей `sklyar_ds_service_price`. Здесь можно задавать врачу услугу, формат приема, локацию, стоимость и длительность приема.
- `Записи на прием` - управление таблицей `sklyar_ds_booking`. Здесь отображаются созданные заявки, а запись можно подтвердить: статус меняется из `new` в `confirmed`.

## Публичный компонент

После установки доступен компонент:

```php
$APPLICATION->IncludeComponent(
    'sklyar:doctorschedule.booking',
    '.default',
    []
);
```

Либо компактный шаблон:

```php
$APPLICATION->IncludeComponent(
    'sklyar:doctorschedule.booking',
    'compact',
    []
);
```

Публичный компонент позволяет:

- выбрать специализацию, врача, формат приема, услугу, дату, локацию и слот;
- открыть форму записи;
- заполнить ФИО, телефон, email и комментарий;
- подтвердить согласие на обработку персональных данных;
- создать запись в таблице `sklyar_ds_booking`.

## Удаление

При удалении модуля удаляются:

- таблицы модуля;
- HL-блоки модуля;
- публичный компонент `sklyar/doctorschedule.booking`;
- административные файлы модуля.

## Назначение текущей версии

Текущая версия предназначена для тестового задания, демонстрации установки модуля и проверки базового пользовательского сценария записи на прием. Для рабочего проекта потребуются дополнительные доработки по безопасности, юридической фиксации согласий, правам доступа, обработке ошибок и интеграции с реальными процессами клиники.
